# IslandTest 任务对齐与当前改进方向

## 这轮已经补齐的内容

### 1. 数据集构建 - 清洗 - 评估
- 新增 `dataset_pipeline.py`
- 支持随机参数采样生成高度图数据集
- 支持基于结构质量的清洗
- 支持数据集统计评估
- 支持直接导出训练用数组

### 2. 特征提取与统一归一化
- 新增 `feature_processing.py`
- PCG 参数统一归一化到 `[-1, 1]`
- 结构指标统一缩放到同一尺度
- latent 特征支持拟合后标准化
- 环境状态与动作空间都改为归一化空间

### 3. 奖励函数与离线评估统一
- 新增 `map_scoring.py`
- 奖励函数不再只服务 RL
- 同一套评分器同时服务：
  - 数据集质量评估
  - 地图大小/结构直接评估
  - RL reward
- 新增 `land_ratio` 指标，显式评估地图大小

### 4. 环境与 baseline 对齐
- `rl_environment.py` 改为归一化动作更新
- 支持可选 VAE latent 接入
- novelty 使用归一化后的特征
- `cmaes_baseline.py` 也改为复用统一评分和归一化参数空间

### 5. 基础验证
- 重写 `test_modules.py`
- 覆盖生成、评估、数据集、归一化、VAE、环境、CMA-ES

## 目前项目的主要改进方向

### 优先级最高
1. 先真实跑一版数据集构建与清洗，拿到统计结果。
2. 用清洗后的数据训练 VAE，并导出 latent 分布统计。
3. 用这批 latent 统计拟合归一化器，再接 RL。

### 中优先级
1. 检查 latent 是否保留原始基础信息。
2. 做插值、重建误差、聚类/分散度分析。
3. 对比“只用结构指标”和“latent + 结构指标”两种状态表示。

### 后续 RL 阶段
1. 先从 PPO 跑通小成本实验。
2. 再切 SAC 做连续动作空间优化。
3. 重点调奖励项权重和动作步长。

## 你现在可以直接往下接的顺序
1. 用 `dataset_pipeline.py` 先生成一批数据并看清洗统计。
2. 用清洗后的高度图训练 `BetaVAE`。
3. 用 `encode_heightmaps(...)` 提 latent，拟合 `IslandFeatureNormalizer`。
4. 把这个 normalizer 和 VAE 传进 `IslandGenerationEnv`。
5. 先跑 PPO，再考虑 SAC。
